SECTION 1

Google Apps Script（GAS）とは？

🎯 Googleをあなたの指先で自動化するツール

Google Apps Script（通称：GAS/ギャス）は、Googleが提供する、無料の自動化プログラミングツールです。 世界中で広く使われている「JavaScript」という言葉を使い、Gmail、スプレッドシート、Googleカレンダーなどを繋いで自動で仕事をこなしてくれます。

RUNNING ENVIRONMENT Cloud Run

サーバーの準備もパソコンへのインストールも必要なし。ブラウザさえあれば準備完了です。

✨ 主な4つの特徴

🍵

完全無料、制限なし

Googleアカウントを持っていれば、誰でも今すぐに無料で使い始めることができます。

💻

インストール不要

Excelマクロ（VBA）のように端末環境に依存せず、ブラウザだけで動き、保存も全自動です。

☁️

自動安全セーブ

作成したコードは自動的にGoogleドライブに安全に保管されます。紛失の心配もありません。

🌲

JavaScriptがベース

Web業界標準のプログラミング言語が使われているため、ネットに役立つ情報がたっぷり溢れています。

💡 GASで可能になる「毎日のスマートな時間」

毎日のタスクを完全自動化 (毎朝決まったメールを確認、定期報告)
サービス同士を優しく連結 (問い合わせフォームからスプレッドシートへ自動入力)
自分だけの計算ルール（関数）を創る (スプレッドシートにオリジナルの計算を追加)
シンプルな申請Webアプリを構築 (社内やサークル用のフォームと連動システム)
SECTION 2

基本概念と最初のコード

プログラミングを始める前に、基本となる少数の言葉を仲間に迎え入れましょう。これだけで全体のイメージがぐっとつかみやすくなります。

🎁 関数（かんすう）

命令をひとつに束ねた「プログラムのレシピ」です。function myName() { ... } のように分かりやすい名前をつけてあげます。

⏱️ トリガー

「毎日朝8時になったら」「フォームが届いたら」など、プログラムを自動的に叩き起こして動かす仕組みのことです。

📑 ログ & デバッグ

「ログ」はスクリプトからの日記のような実行記録。「デバッグ」は、その記録を見てエラーの原因（虫＝バグ）を取り除く作業です。

🚀 デプロイ

完成したスクリプトを「一般公開して、他の人や外部システムからも呼び出せる状態」に準備してあげる手続きです。

myFirstFunction.gs JavaScript 
function myFirstFunction() {
  // Logger.log は、実行ログに文字を出力する、最も基本的な命令です
  Logger.log("Hello, Google Apps Script!");
}

この関数の動かし方

  1. 上部のツールバーから実行したい関数 「myFirstFunction」 を選択。
  2. 「▶️ 実行」 ボタンをクリック。
  3. 画面の下に「実行ログ」がスッと開き、数秒後に Hello, Google Apps Script! と表示されれば無事大成功です！

左サイドバー

スクリプト全体の「モード切替スイッチ」です。ファイル管理、自動で動く「トリガー」の設定、過去の実行ログを見る画面などへ、いつでもここからひとっ飛びできます。

ファイルパネル

プロジェクトに入っているすべての引き出し（プログラムファイル）です。コードが長くなってきたら、utilities.gsのようにファイルを分割してきれいに整理整頓しましょう。

上部ツールバー

スクリプトを実行するための「指揮棒」です。動かしたい特定の関数を選択し、横の「▶️ 実行」を押すだけで、プログラムが即座にパタパタと走り出します。

コードエディタ

メインとなる「キャンバス（作業机）」です。ここにあなたの言葉でレシピ（関数）を書いていきます。文字の色分け（シンタックスハイライト）や、エラーを自動で赤線で警告するサポート機能が付いています。

SECTION 4

左サイドバーの使い方

左サイドバーをうまく活用することで、プログラムのデバッグ（見直し）や、自動実行をすべて手中で管理できるようになります。

📁 ファイル管理（◀▶ パネル）

新しく「.gs」スクリプトを作ったり、Web用の「.html」を追加してデザインUIを作ったりできます。

📂 プロジェクト整理の目安
・main.gs : 本番のメール・転記処理
・utilities.gs : 日付計算や共通の部品
🕐 実行履歴

過去に実行されたトリガーや手動スクリプトの成否を、リストで一覧確認。エラーが出た際、ここが一番の調査拠点になります。

✓ 成功 (10:30) - checkEmails() 2.5s
✗ 失敗 (10:25) - sendMail() Error: defined
⏱️ トリガー管理（自動実行の魔法）

トリガーを設定することで、あなたの代わりにGASが24時間体制で、完全放置・自動で働いてくれるようになります。

⏰ 時間ベーストリガー

「毎週月曜日の朝9時」「1時間ごと」のように、決まった時間・スケジュールで自動運転させます。

🎯 イベントベーストリガー

「Googleフォームに回答が送信されたとき」「スプレッドシートの値を編集したとき」に即時に稼働します。

≡ プロジェクト設定 & 鍵（API KEY）の保護

「パスワードや外部APIの認証キー」はコード内に直接書くと、他人に誤って共有した時に漏洩してしまいます。 そこで、安全に秘密の鍵を隠すための引き出しが 「スクリプトプロパティ」 です。

// スクリプトプロパティから安全にAPIキーを取得する例
const props = PropertiesService.getScriptProperties();
const apiKey = props.getProperty("API_KEY");
SECTION 5

上部ツールバーの使い方

ツールバーは、あなたが書いたプログラムを優しく手助けする道具箱です。

▶️

実行（Run）

選んだ関数をその場でスタート。プログラムに命が吹き込まれる瞬間です。

🐛

デバッグ（Debug）

コードの途中（ブレークポイント）で一時停止させ、1行ずつ変数の中身を確認しながら細かく動かす高度な診断機です。

📊

実行ログ（Logs）

Logger.log() で送られた「処理の経過メモ」を確認できます。何件メールを送ったかなど、進捗を教えてくれます。

SECTION 6

暮らしと仕事を支える3つの実践例

実際に明日から職場で役立つGASスクリプトのひな形をご用意しました。 ハーブの温かいお茶を選ぶように、使いたいレシピを選んでエディタへ連れて帰ってください。

🎯 毎朝8時に「重要」メールが溜まっているかチェックして超過通知

function checkEmailsAndNotify2() {
// 1. Gmailから「重要」ラベルの付いたスレッドを検索
const threads = GmailApp.search('label:重要');
// 2. スレッド数（メール件数）をログに保存
Logger.log("重要な未処理メール: " + threads.length + "件");
// 3. もし5件以上溜まっていたら、あなたのアドレスへ通知
if (threads.length >= 5) {
GmailApp.sendEmail(
"your-email@example.com",
"【GAS警告】重要なメールがたまっています",
"重要な未処理メールが" + threads.length + "件に達しました。確認しましょう！"
);
}
}
🕰️ トリガー登録のススメ：

エディタの左メニュー「⏱️」を開き、この「checkEmailsAndNotify2」を選択。実行間隔を「時間ベース ＞ 日ごと ＞ 午前8時〜9時」で保存すれば、パソコンを閉じていても毎朝自動でGmailをパトロールしてくれます。

🎯 セルの中で「=CELSIUS_TO_FAHRENHEIT(A1)」と入力して温度を変換

/**
* 摂氏(℃)を華氏(℉)に自動で変換するカスタム関数です
* @customfunction
*/
function CELSIUS_TO_FAHRENHEIT(celsius) {
// 摂氏の数値を引数で受け取り、計算式に入れて返却
return (celsius * 9/5) + 32;
}
🍀 スプレッドシート側の使い方：

1. スプレッドシートを開き、セルに 0 を入力（例：セルA1）。

2. 隣のセルに =CELSIUS_TO_FAHRENHEIT(A1) と入力してエンター。

3. するとセルに 32 と表示されます。あなた専用のオリジナル計算関数の出来上がりです！

🎯 Googleフォーム送信直後にお客さまに自動お礼メールを自動配信

function onFormSubmit(e) {
// 1. 送信イベントデータ(e)から送信したユーザーのメールを取得
const email = e.response.getRespondentEmail();
// 2. 即座にお客さまにお礼のメールを自動配信
GmailApp.sendEmail(
email,
"【受付完了】フォームへのご送信ありがとうございました",
"お世話になっております。\n\nお送りいただいた内容を受付いたしました。\n確認後、担当者より改めてご連絡申し上げます。"
);
}
💡 動かすための大切なトリガー条件：

フォームの自動返信を動かすには、トリガー設定で「イベントのソース：スプレッドシート（またはフォーム）」「イベントの種類：フォーム送信時」に設定します。これだけで、昼夜を問わず瞬時にお礼が届く親切な対応が完成します。

SECTION 7

よくある困りごと・解決方法

プログラムを書いているときに少し立ち止まったら、このレスキューボックスを開けてみてください。

まずはツールバーの「📊 実行ログ」や左サイドバーの「🕐 実行履歴」に表示されている英文をそっと見つめてみましょう。よくある原因と解決方法です：

表示される主なエラー エラーが起きる原因 やさしい解決ヒント
ReferenceError: xxx is not defined 変数名や関数名のスペルミス、未定義 スペルが大文字・小文字も含めて合っているか再チェックします。
TypeError: Cannot read properties of undefined 指定したファイル名、シート名が見つからない スプレッドシートのシート名「シート1」などが完璧に一致しているか確認します。
Exception: Authorization required Googleサービスへのアクセス権限未承認 初回実行時に画面に出る「アクセスを承認（詳細＞安全ではないページに移動）」を許可します。

ご安心ください。これは新しく作ったプログラムを認証するための正常なプロセスです。

あなたが作成した新しいスクリプトは、Googleにとって「まだ審査されていない独自のプログラム」であるため、注意のために警告画面（このアプリは確認されていません）を表示します。

【進め方】：
「詳細」 ＞ 「〇〇（プロジェクト名）に移動（安全ではない）」をクリックし、アカウントへのアクセス要求を「許可」して進めれば、二回目からは警告が出ず正常に動作するようになります。

通常の個人利用・事務レベルの自動化であれば、無料枠（クォータ制限）内でゆったりと間に合うことがほとんどです。主な制限値は以下のとおりです：

  • 1日のスクリプト実行合計時間： 6時間 / 日（有料プランWorkspaceなら24時間）
  • 1回の連続稼働の上限： 最大30分（それ以上の超長時間処理は分割する必要があります）
  • メール送信件数制限： 100通 / 日（Workspaceなら1,500通 / 日）

はい、自動で完璧に連携されます。

GASエディタのファイルパネルで main.gs や utilities.gs のように複数ファイルに分けて格納しても、実行時にはそれらはひとつのプログラムとして結合して認識されます。

ですので、面倒なインポート命令などを書かなくても、別のファイルに書いた関数をまるで同じファイル内にあるかのようにスムーズに呼び出すことができます。

SECTION 8

学習ロードマップと次のステップ

一歩ずつ成長を楽しめる、初心者から中級者へ向けた穏やかなロードマップです。

初級（1〜2週間）：まず動かしてみる楽しさを知る

・まずはスプレッドシートから拡張機能 ＞ Apps Scriptをクリック。
・Logger.log() で文字を表示させたり、簡単な足し算を返す関数を動かして「自分で書いたものが動く楽しさ」にふれましょう。

中級（2〜4週間）：自動化とイベントトリガーに挑む

・「⏱️ トリガー設定」をマスターして、時間による自動実行や、フォーム回答時の通知を動かしてみましょう。
・Gmail、カレンダー、スプレッドシートのデータをスムーズに連携して書類作成の時間を削減します。

上級（1ヶ月以上）：Webアプリや外部APIと繋ぐ

・LINE Notify や Slack と連携して、スマホに通知を飛ばすシステムを作ります。
・外部の天気予報情報や株価などのAPI情報、あるいは自社のデータベースからデータを自動収集する独自のWebサイトを作れるようになります。
